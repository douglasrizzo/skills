# skills

Personal rules and skills for AI coding agents (Claude Code, Codex, and Cursor),
focused on software projects. Includes Python-specific idiom and tooling rules for
Python/ML codebases.

Engineering standards apply to every session. Skills are on-demand workflows —
feature implementation, code review, testing, debugging, and pull request management —
that the agent follows step by step when invoked.

## How it works

Content lives in `fragments/` as individual Markdown files. `scripts/build.sh`
assembles them into per-agent output files in `dist/`. The installer symlinks from
`dist/` to each agent's global config directory.

Each agent gets exactly what it needs — different RTK usage models, Context7 placement,
and frontmatter formats — with no `@include` dependencies in the generated files.

## Structure

```
fragments/                           # Human-authored source components
  universal.md                       # Language-agnostic core (all agents)
  python.md                          # Conditional Python section (all agents)
  rtk-claude.md                      # RTK: Claude Code hook model
  rtk-codex.md                       # RTK: Codex manual prefix model
  context7.md                        # Context7 MCP instructions
  frontmatter-claude.md              # YAML frontmatter for Claude Code
  frontmatter-cursor-standards.md    # YAML frontmatter for Cursor standards rule
  frontmatter-cursor-context7.md     # YAML frontmatter for Cursor context7 rule
dist/                                # Generated by build.sh — do not edit directly
  CLAUDE.md                          # → ~/.claude/CLAUDE.md
  AGENTS.md                          # → ~/.codex/AGENTS.md
  engineering-standards.mdc          # → ~/.cursor/rules/engineering-standards.mdc
  context7.mdc                       # → ~/.cursor/rules/context7.mdc
rules/
  python-examples.md                 # On-demand before/after idiom code examples
scripts/
  build.sh                           # Generates dist/ from fragments/
  install-skills.sh                  # Runs build.sh, then symlinks to agent locations
skills/
  audit-mutmut/                      # Run mutmut and add tests to kill survivors
  bootstrap-python-project/          # Scaffold a new Python repo
  commit/                            # Atomic commits with pre-commit
  create-jira-ticket/                # Draft a Jira ticket from branch contents
  create-plan/                       # Create a multi-phase .plans/ directory
  debug-code/                        # Diagnose errors, NaNs, shape mismatches, CUDA
  design-patterns-ml/                # When and how to apply patterns in ML codebases
  find-dead-and-duplicate-code/      # Report dead code and duplicate logic
  generate-hypothesis-tests/         # Generate and refine property-based tests
  git-new-branch/                    # Create a branch with optional commit + push
  implement-feature/                 # Plan, place, DRY-check, implement, validate
  implement-tests/                   # Add or extend tests with ML-specific guidance
  next-phase/                        # Implement the next phase from .plans/
  open-pr/                           # Draft and open a pull request via gh
  review-code/                       # Review existing code for quality and correctness
  review-plans/                      # Reconcile .plans/ with current repo state
  review-pr/                         # Fetch and review an open pull request via gh
  rewrite-branch-commits/            # Clean up a branch's commit history
```

## Installation

Clone this repo to a stable location. Run the installer from the repository root:

```bash
./scripts/install-skills.sh claude
./scripts/install-skills.sh codex
./scripts/install-skills.sh cursor
```

The installer runs `build.sh` automatically, then symlinks the generated files and
skills to the selected agent's global config directory.

If you want commits to keep generated outputs in sync automatically, enable the
included pre-commit hook:

```bash
pre-commit install
```

The hook rebuilds `dist/` whenever files under `fragments/` or `scripts/build.sh`
change.

GitHub Actions also rebuilds `dist/` on pushes and pull requests and fails if the
generated files are out of date.

### Claude Code (global, one-time)

```bash
./scripts/install-skills.sh claude
```

Engineering standards are injected into every session. Skills become slash commands:
`/implement-feature`, `/review-code`, `/debug-code`, etc.

### Codex (global, one-time)

```bash
./scripts/install-skills.sh codex
```

Installs `~/.codex/AGENTS.md` (engineering standards + RTK + Context7), skills under
`~/.codex/skills/`, and `python-examples.md` under `~/.codex/rules/`.

### Cursor (global, one-time)

```bash
./scripts/install-skills.sh cursor
```

Installs two rule files and all skills:
- `~/.cursor/rules/engineering-standards.mdc` — always-on core + Python section
- `~/.cursor/rules/context7.mdc` — Context7 MCP instructions
- `~/.cursor/skills/<skill>/` — one directory symlink per skill

If `python-engineering-standards.mdc` exists from a previous install, it is removed
automatically.

### Cursor (per-project)

To wire skills into a specific project's `.cursor/` directory instead:

```bash
SKILLS_ROOT=$(cd /path/to/this/repo && pwd -P)
cd /path/to/your/project

mkdir -p .cursor/rules .cursor/skills

ln -sf "$SKILLS_ROOT/dist/engineering-standards.mdc" \
       .cursor/rules/engineering-standards.mdc
ln -sf "$SKILLS_ROOT/dist/context7.mdc" \
       .cursor/rules/context7.mdc

for dir in "$SKILLS_ROOT"/skills/*/; do
  [ -d "$dir" ] && [ -f "$dir/SKILL.md" ] || continue
  name=$(basename "$dir")
  target=.cursor/skills/"$name"
  if [ -L "$target" ]; then
    rm -f "$target"
  elif [ -d "$target" ]; then
    rm -rf "$target"
  fi
  ln -sfn "${dir%/}" "$target"
done
```

### Updating after fragment changes

Re-run the installer to rebuild `dist/` and refresh symlinks:

```bash
./scripts/install-skills.sh claude   # or codex / cursor
```

To rebuild `dist/` without reinstalling:

```bash
./scripts/build.sh
```

## Usage

### Claude Code

Invoke a skill with its slash command:

```
/audit-mutmut                  /bootstrap-python-project /commit
/create-jira-ticket            /create-plan              /debug-code
/design-patterns-ml            /find-dead-and-duplicate-code
/generate-hypothesis-tests     /git-new-branch
/implement-feature             /implement-tests          /next-phase
/open-pr                       /review-code              /review-plans
/review-pr                     /rewrite-branch-commits
```

The agent will follow the selected skill's workflow. Some skills are advisory and some
perform implementation or repository operations.

### Cursor

With the symlinks above, the always-on rule applies every chat. Workflows are available
as agent skills under `~/.cursor/skills/` (or `.cursor/skills/` per project). The model
selects them when your task matches their description, or you can name a skill in your
prompt (e.g. "follow the implement-feature skill").

## Design philosophy

- Engineering standards encode what should always be true (architecture, naming, patterns).
- Skills encode workflows that require judgment and user interaction.
- Design patterns are applied reactively to complexity, not preemptively to every file.
- `fragments/` is the source of truth; `dist/` files are assembled outputs.
